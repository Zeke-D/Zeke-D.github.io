Connected

Tech Specs/History

Connected actually started as an idea between my cousin and I to create an app that got people together in groups in the real world. Over the past year the idea got thrown back and forth, hashed and rehashed, scrapped and restarted until we finally decided to start developing it. I also used this opportunity to teach my cousin how to code.

I used React Native (including ES2019 standards, JSX, JavaScript, styling via CSS) to develop the app. The backend is ran with NodeJS and Express. We are using MongoDB for database management and are currently implementing OAuth 2.0 Authentication standards.

Mobile Development

When I started this project, I really wanted a way to learn more about mobile development. I thought initially I wanted to use Swift and make a strictly iOS app. And while there is a lot of power in using a language normalized for a specific architecture, like Apple's Swift or Objective-C, it felt both:

  1. Very limiting to my intended user-base
  2. Like a sledgehammer to drive in a nail
With those languages, if I wanted Android users, I'd have to redevelop the app in Kotlin or something, and that sounded like a real pain. Instead, I decided to explore this newer language I had heard about called React Native. React Native is basically a way to program apps using React-style syntax and have it compile down to run natively on whatever machine was running it. It sounds awesome, right? You never have to develop twice again. But do note there are some serious drawbacks:
  1. You have very little performance control.
  2. The app is rendered in a container, which limits the interaction you have with the hardware.
  3. It can be a little "block-boxy", in that there is not a whole lot of transparency.
With that being said, I didn't really need high performance or to talk to the hardware specifically, so this was a perfect use-case for react-native.
